# US-01.03 Log Out

### User Story ID
`US-01.03`

### User Story Name
Log Out

### User Story
As a **user**, I want to log out, so that my account stays secure.

---

### Pre-conditions
| Pre-condition ID | Description |
|------------------|-------------|
| PR01 | The user is currently authenticated (has an active session). |
| PR02 | A Log Out action is accessible from the authenticated UI (e.g., navigation menu). |

---

### Workflow/Activity Diagram

1. Authenticated user selects "Log Out" from the navigation.
2. System invalidates the current session and all other active sessions for that user across browser tabs.
3. System clears the local auth token/cookie.
4. User is redirected to the public home page (visitor view).

**Alternative flow — expired session:**

1. User attempts to trigger Log Out when their session has already expired.
2. System completes the logout action silently (no error shown).
3. User is redirected to the public home page.

---

### Suggested UI Specifications

| Field Name | Type | Mandatory | Default Value | Description |
|---|---|---|---|---|
| Log Out button / link | Navigation action | Yes | N/A | Visible in the navigation menu for all authenticated users. Triggers the logout flow on click. No confirmation dialog required. |

**Post-logout state:**

| Element | Behaviour |
|---|---|
| Browser tab after logout | Redirects to the public home page. Other open tabs also lose authenticated state on next interaction or page reload. |
| Back button after logout | Navigating back must not restore the authenticated session. Cached authenticated pages should not be served. |

---

### Business Acceptance Criteria

**AC01** — Successful logout

**Given** an authenticated user selects Log Out,
**When** the action is confirmed,
**Then** the system invalidates the session, clears local auth state, and redirects the user to the public home page (visitor view).

**AC02** — All-session invalidation

**Given** a user is logged in across multiple browser tabs,
**When** they log out from any tab,
**Then** all active sessions for that user are invalidated. Other open tabs lose authenticated access on the next interaction or page reload.

**AC03** — Expired session — silent logout

**Given** a user's session has already expired,
**When** they trigger the Log Out action,
**Then** the system completes the logout silently without displaying an error, and the user is redirected to the public home page.

**AC04** — Post-logout navigation guard

**Given** a user has logged out,
**When** they navigate back in the browser or attempt to access an authenticated route directly,
**Then** they are redirected to the Log In page. The previous authenticated session is not restored.

---

### Out of Scope
| OOS ID | Description |
|--------|-------------|
| OOS01 | Confirmation dialog before logout — logout happens immediately on user action. |
| OOS02 | "Log out from all devices" as a separate explicit action — all-session invalidation happens automatically on every logout. |

---

### Non-functional Requirements
| Requirement | Description |
|---|---|
| Security | Session tokens must be fully invalidated server-side on logout, not just cleared client-side. The browser must not be able to reuse an invalidated token. |
| Performance | Logout should complete and redirect within 1 second. |
| Cross-platform | Logout must function correctly on mobile and desktop browsers (Chrome, Safari, Edge — latest two versions). |

---

### Dev Handoff Notes

- Auth provider: Supabase Auth (Phase 1). Use `signOut({ scope: 'global' })` to invalidate all sessions, not just the current tab.
- Clear any local state (e.g., Zustand store, React context, cookies) on logout to prevent stale UI.
- Implement route guards on all authenticated routes: redirect unauthenticated requests to the Log In page. Use `Cache-Control: no-store` or equivalent headers on authenticated pages to prevent browser back-button restoration.
- Phase 2 (Azure AD): session invalidation will use AAD sign-out endpoint. The application redirect logic is unchanged.
