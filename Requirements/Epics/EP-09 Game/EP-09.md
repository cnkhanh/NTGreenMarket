# EP-09 — Lucky Draw

## Goal
Run a lucky draw during the event to reward participants and drive engagement. Entries are earned through event participation (purchases, donations, or both — TBD).

## In Scope
- Employees earn lucky draw entries by participating in the event (buying, donating — entry rules TBD)
- View how many entries you have
- Admin: configure draw rules (how entries are earned, number of prizes)
- Admin: run the draw and announce winners
- Winners displayed in-app (results page or banner)

## Out of Scope
- Real-time animated draw (TBD — could be a nice Gen Z touch)
- Prize fulfilment / logistics (handled offline)
- Multiple draw rounds (single draw per event — TBD)

## Entry Rules (TBD — confirm with stakeholders)
- Option A: 1 entry per order placed
- Option B: 1 entry per donation made
- Option C: entries earned by both orders and donations
- Option D: tiered entries based on amount spent/donated

## Roles Involved
- Employee (Buyer)
- Admin

## Design Notes
- Entry count should be visible and motivating throughout the app (nudge participation)
- Results announcement moment should feel exciting — consider a simple animation
